Tunaar on Docker

A single-container bridge that makes IPTV playlists and real HDHomeRun tuners appear in Plex as one Live-TV device — with a unified guide.

What Tunaar does

📺

HDHomeRun emulation

Answers Silicondust discovery so Plex, Emby & Jellyfin auto-find it as a network tuner.

🔗

M3U ingestion

Pulls one or more IPTV playlists and merges them into a single, numbered lineup.

📡

Real HDHomeRun ingest

Absorbs a physical HDHomeRun's channels so Freeview/OTA and IPTV live behind one device.

🗓️

XMLTV guide

Serves a merged guide, auto-detected from playlists, with name-based matching for channels lacking an ID.

🎞️

ffmpeg remux

Remuxes streams for reliable Plex playback, with direct-passthrough and redirect modes too.

⚡

One-click country playlists

Add free iptv-org country lists (UK, US, Canada, France, Germany, Spain and more) in a click.

🩹

Plex duplicate-description fix

Optional guide tweak so Plex keeps a separate description per airing — clean titles, date-based episode numbers.

🎛️

Web dashboard

Manage sources, guide, groups and tuners from a branded UI — no config files needed.

🔄

Auto-update

Optional Watchtower companion keeps the container current, scoped to Tunaar only.

Requirements

  • Docker on a host that's always on — a NAS (QNAP/Synology/Unraid), mini-PC, or home server.
  • Plex Media Server (or Emby / Jellyfin) on the same LAN.
  • An IPTV playlist URL and/or a real HDHomeRun device — you can also start empty and add sources from the dashboard.
  • A few hundred MB of disk for the image; a small persistent volume for config.
Tunaar runs best with host networking so Plex's HDHomeRun auto-discovery (UDP 65001) works. On platforms without host networking, use bridge mode and add the tuner by IP.

Install

Both options produce the same running container. Tunaar is distributed as a Docker image — there's nothing to clone or build.

Option A — Plain docker run (recommended)

docker run -d --name tunaar --restart unless-stopped \
  --network host \
  -v /share/Container/tunaar/config:/config \
  ghcr.io/munerisuk/tunaar:latest

On a host without host networking, swap --network host for -p 5004:5004. Then browse to http://<host-ip>:5004.

Option B — Docker Compose

services:
  tunaar:
    image: ghcr.io/munerisuk/tunaar:latest
    container_name: tunaar
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./config:/config
docker compose up -d

Configuration & environment

Everything is editable in the dashboard, but these env vars seed the container or override config:

VariableDefaultWhat it does
TUNAAR_PORT5004Dashboard / API port (bridge mode only).
TUNAAR_PLAYLIST—Optional IPTV M3U URL to seed on first run.
TUNAAR_EPG_URL—Optional XMLTV guide URL.
Persist your config — important. Always mount /config to a named volume (e.g. -v tunaar-config:/config) or a host path. If you run the container with no /config mapping, Docker creates an anonymous volume that is not carried over when the container is recreated — so an update (especially via Watchtower) starts with a blank config and you'll have to re-add your sources, guide and channel mappings. Tunaar warns on the dashboard if it detects this.

Automatic updates

Two ways to stay current — pick one.

Hands-off — Watchtower (recommended)

A tiny companion container checks for new images and updates Tunaar for you. Scope it to only the tunaar container so nothing else on your host is touched.

docker run -d --name watchtower-tunaar --restart unless-stopped \
  -v /var/run/docker.sock:/var/run/docker.sock \
  containrrr/watchtower --cleanup --interval 86400 tunaar

Lower --interval (seconds) to check more often; stop it with docker rm -f watchtower-tunaar.

One-click from the dashboard

The Console → Update button lets Tunaar pull a new image and recreate itself. Enable it by adding the Docker socket and flag to your docker run / Compose:

-e TUNAAR_SELF_UPDATE=1 \
  -v /var/run/docker.sock:/var/run/docker.sock
Heads up: both grant Docker (root-equivalent) access — to the Watchtower image, or to the Tunaar container itself. Opt in deliberately. Prefer neither? Pull the new image and recreate the container over SSH.

Verify it's working

  • Dashboard loads at http://<host-ip>:5004 and shows status cards.
  • Health check: docker ps shows the container healthy (or curl http://<host-ip>:5004/healthz).
  • Tuner endpoints respond: /discover.json, /lineup.json, /epg.xml, /device.xml.
