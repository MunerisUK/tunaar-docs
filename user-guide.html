Using Tunaar

Tunaar is installed and running. This guide walks through adding channels, getting a guide, and connecting it to Plex — all from the web dashboard.

Getting started

Open the dashboard in any browser on your network:

http://<your-host-ip>:5004

That's the only screen you need — every setting lives here. If it doesn't load, check the container is up (docker ps) and you're using the host's LAN IP.

Setup wizard (easiest start)

On a fresh install the Setup wizard opens automatically and walks you through everything in four steps. You can re-run it anytime from the Setup wizard link in the top nav.

  1. Add channels

    Click the free-provider presets you want, and/or add your own M3U list or a real HDHomeRun.

  2. Guide / EPG

    Leave auto-detect on; add an extra XMLTV only if a source needs its own guide.

  3. Trim groups

    Untick what you don't watch. A live counter warns (turns red) if the lineup is too big for Plex — aim for a few hundred.

  4. Connect to Plex

    Copy the tuner and /epg.xml URLs straight from the final step into Plex's DVR setup.

Prefer to do it manually? Every wizard step maps to a panel under Settings — the sections below cover each one.

Adding channels

Open Settings → Sources. You can mix as many sources as you like — they merge into one numbered lineup.

  • Free provider presets — one click, guide included. See below.
  • M3U playlist — set type to M3U playlist, paste the playlist URL. Optionally set a name and a "Force group".
  • Real HDHomeRun — set type to HDHomeRun, enter the device address. See below.
The optional Force group field overrides every channel's category from that source — handy for keeping providers tidy in Plex (e.g. all of one source's channels under "Sports"). Use Max ch. to cap how many channels a source contributes (great for taming huge lists at the source).
Adding an Xtream-style playlist (…/get.php?username=&password=)? Tunaar auto-detects the matching xmltv.php guide and adds it for you — no need to find the EPG URL.

Free country playlists (one click)

Under Sources you'll find a Quick add free country playlists row — one button per country (UK, US, Canada, France, Germany, Spain, Italy, Brazil, Australia, Mexico, Jamaica). Click to add that country's free streams; added ones show a ✓, and clicking again removes them.

These come from the iptv-org community project — publicly-indexed streams, maintained by volunteers. Coverage and reliability vary, and they don't include a guide, so add an XMLTV guide below and let name-matching fill it in. A dead source is skipped automatically, not fatal.

Adding a real HDHomeRun

Have a physical HDHomeRun (e.g. for Freeview/OTA)? Let Tunaar absorb it so Plex sees one tuner with both OTA and IPTV — and a single guide.

  1. Find the device URL

    In a browser, confirm http://<hdhr-ip>/lineup.json loads. Use http://<hdhr-ip> as the source URL.

  2. Add it

    Settings → Sources, type HDHomeRun, paste the address. Its channels appear in the lineup (default group "Freeview", keeping their channel numbers).

  3. Clean it up (optional)

    Freeview tuners report a lot of non-TV entries — radio stations, shopping, adult and +1 timeshifts. Tick Clean lineup when adding the HDHomeRun and Tunaar skips those, leaving a tidy TV-only set.

  4. Get them a guide

    OTA channels have no guide ID, so Tunaar matches them by name to your XMLTV. Add a guide for your region (below) and watch the EPG match count rise.

Don't also add the HDHomeRun directly in Plex — let Tunaar front it, or you're back to Plex's "can't mix guide types" limitation.

Guide / EPG

Open Settings → EPG.

  • Auto-detect (on by default): uses the guide URL advertised inside each playlist — most providers include one, so the presets "just work".
  • Quick-add guides: one-click buttons for regional XMLTV guides (UK/US/CA/FR/DE).
  • Manual URLs: paste extra XMLTV guide links (one per line), e.g. a UK Freeview guide for OTA channels.
  • Name matching: channels without a guide ID are matched to guide entries by name (HD/quality differences ignored), so OTA/HDHomeRun channels get data automatically.
  • Fix Plex duplicate-description bug (optional): see below.

Tunaar serves the merged result at /epg.xml — that's the single URL you give Plex.

Paste one URL per line (Tunaar also splits multiple URLs pasted on one line). If a source is unreachable it's skipped with a notice rather than wiping the whole guide, and guides are fetched with a browser user-agent so hosts that block bots still work.

Fix Plex duplicate-description bug

Plex's DVR keys programme descriptions by title, so airings that share a title — classic case: "MLB Baseball", "Premier League" on several channels — all end up with the same description. It's a Plex parser bug, not a guide problem.

Tick Fix Plex duplicate-description bug in the EPG panel and Tunaar makes each airing distinguishable in the guide it serves Plex, two ways depending on the programme:

  • Live events with a sub-title (sport especially) → the sub-title is folded into the title, e.g. "Football — Arsenal v Spurs" vs "Football — Liverpool v Everton".
  • Everything else (news, paid programming, music, generic titles) → the title is left clean and a date-based episode-num is injected from the airing date instead. The season is the year and the episode is the Julian day-of-year, so 27 June 2026 becomes S2026E178 (onscreen) / 2025.177.0 (xmltv_ns). A show that airs more than once that day gets a trailing index — S2026E178.2, .3 — so every airing is unique.
It's off by default because it changes the guide Plex sees. Episodic content and films are left alone: anything with a season/episode (episode-num) tag is skipped since Plex separates those itself, and movies (categories like Movie / Feature Film) are skipped too — a movie's description is the same across airings and its release-year <date> already distinguishes it, so injecting an episode number would wrongly make Plex treat it as a TV series. So "News" and "Paid Programming" stay clean while Plex still keeps a separate description per airing.

Manual EPG mapping

Some channels won't match the guide automatically — most often regional Freeview from a real HDHomeRun, where the tuner reports "BBC One South" but the guide calls it "BBC One". Plex can't fix this (it won't mix its postcode guide with XMLTV in one setup), so Tunaar lets you map them by hand.

  1. Load a guide first

    The picker is populated from the channels in your loaded guide, so make sure your EPG is working (a real programme count on the dashboard) before mapping.

  2. Open the panel

    Settings → Manual EPG mapping. Leave Unmatched only ticked to see just the channels with no guide. Use the search box to find one fast.

  3. Pick the guide entry

    Start typing in a channel's box and choose the matching guide channel from the dropdown (shown as Name · id). It saves instantly and is marked (manual).

  4. Done — it sticks

    Manual mappings are saved and always win over auto-matching. Hit Clear on a row to remove one. Refresh the Plex guide afterwards.

Tip: map the mainstream channels you actually watch — you don't need to map every obscure one. Trim the rest with Groups.

Groups (show / hide)

Open Settings → Groups. Every group is shown by default — untick any you don't want in Plex. Great for trimming a huge US list down to the categories you actually watch. Click Save groups when done.

Add the tuner in Plex

  1. Open DVR setup

    Plex → Settings → Live TV & DVR → Set up Plex DVR.

  2. Pick Tunaar

    It should appear automatically as an HDHomeRun tuner. If not, choose "enter address manually" and use http://<host-ip>:5004.

  3. Skip Plex's guide, choose XMLTV

    When asked for guide data, select "Use an XMLTV guide" and enter http://<host-ip>:5004/epg.xml.

  4. Map & finish

    Let Plex match channels to the guide, then finish. Your channels appear under Live TV.

One DVR only. Use a single Tunaar DVR with the XMLTV guide. Don't run a second DVR using Plex's own guide alongside it, or the lineups conflict. Emby/Jellyfin work the same way — add Tunaar as an HDHomeRun tuner and point the guide at /epg.xml.

Stream modes

Under the hood Tunaar can deliver streams three ways. The default suits most setups; change it only if you hit playback issues.

ModeBest for
ffmpeg defaultMost reliable — remuxes the stream so Plex plays it cleanly. Needs a little CPU.
directPassthrough proxy — lower overhead, but the source must already be Plex-friendly.
redirectHands Plex the source URL directly — lowest load, but Plex must reach the source itself.

Updating Tunaar

If you enabled Watchtower at install, you're done — it updates automatically. Otherwise pull the new image and recreate the container (your config lives on the mounted volume, so it's kept):

docker pull ghcr.io/munerisuk/tunaar:latest
docker rm -f tunaar && docker run -d --name tunaar ...   # same flags as install

Or use the dashboard's Console → Update button if you mounted the Docker socket. See the Install Guide for both options.

FAQ

Does Tunaar provide channels?

No. Tunaar is a bridge — you supply the playlists/devices. The presets just point at free, public country playlists from the iptv-org project (UK, US, Canada, France, Germany and more).

What about Plex's own free Live TV?

That's built into Plex itself and needs no tuner — you'll already see it in the Plex app under Live TV. Tunaar is for your playlists and HDHomeRun, which Plex can't show natively.

Can I run IPTV and a real aerial together?

Yes — that's a headline feature. Add both as sources; Plex sees one unified tuner and one guide.

How many channels can I have?

Thousands. Use Groups to hide what you don't watch so Plex stays tidy.
